We will create an ecommerce API in this project.

The following functionalities will be implemented by you: The API will be described in detail further on.

  1. Signing up and logging in as a user. Authentication should be done through JWT.
    The role field in the user schema can have one of two values: "user" or "admin." By default, the role of each new user generated will be "user." So the user schema should have four fields: "fullName," "email," "password" (don't save it as plain text, hash it), and "role," with only the first three being utilised in signup. Verify that the email address is valid, and that the password and fullName are both at least six characters long.

  2. Apart from that we need models for Category, Product, Cart and Order.
  3. Category will inlcude one field "name" which is the name of the category.
  4. Product will have fields for "name," "price," and "quantity," as well as a field for "category," which will store the id of any Category model object.
  5. Cart will include field "user" which stores the id of user who created it.
  6. Cart will also contain a field "products" which is an array of all the products user want to add, which will be of this structure {product:productId,quantity:number}.
  7. Order model  will contain "ordered_by" field which as you can guess contains the id of the user who created it,"items" field which will be an array containing objects of this structure. {product:productId,quantity:number}. "totalPrice" field will contain the order total i.e product.quantity*product.price.
  8. Since every order has a stutus ours will too, field "status" by default its value will be "placed", this will be automatically set when user places the order. If user has no cart associated with them return{message:'No cart found'} with status code 400.
  9. Also while creating Order check if user cart has items which have quantity more than available for that product, if that happens return this&nbsp {message:'Some products are out of stock',outOfStockItems:'Item-names  seperated by space'};

API Spec

  • Only users who are admin can create categories and products.
  • Unauthenticated users can view products and categories,but can not create cart and place order
  • Requests which require authentication must include "auth-token" in header, make your app check in that header for JWT token while doing auth(Create a middleware which you use on each route requiring auth, which checks for "auth-token" in header,checks if its valid or not).
